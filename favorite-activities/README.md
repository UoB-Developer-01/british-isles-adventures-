# Favorite Activities Picker

A single-page vanilla JS/HTML/CSS app for picking, ranking, and sharing favorite activities from `1-activities.md`.

## Files

| File | Purpose |
|---|---|
| `index.html` | Page markup — 3 step panels (Pick / Rank / Share) plus header, footer, toast. |
| `style.css` | Styling, light/dark theme via `prefers-color-scheme`. |
| `app.js` | All app logic (no dependencies, no build step). |
| `activities.json` | Activity data, generated from `1-activities.md`. |

## Running it

`fetch('activities.json')` requires HTTP, not `file://`. Serve the folder locally, e.g.:

```
npx serve favorite-activities
# or
python -m http.server 8080 --directory favorite-activities
```

Then open `http://localhost:<port>/index.html`.

## Data: `activities.json`

Array of objects:

```json
{ "id": "act-001", "name": "Abseiling", "category": "Adventure & Adrenaline Activities", "cost": "£40–£65" }
```

- Generated by parsing every markdown table in `1-activities.md` (one script run, not checked into the repo).
- Deduplicated by activity name (case-insensitive); first category encountered wins.
- The recap sections ("Highest-Rated Experiences", `"Don't Miss" Bucket List`) are parsed last and only contribute activities not already captured elsewhere, tagged under category `"Top Picks"`.
- 296 activities across 17 categories.

To regenerate after editing `1-activities.md`, re-run the same parsing approach (read the md, split on `# ` headers, parse each pipe table, dedupe by name) and overwrite `activities.json`.

## App flow

1. **Pick** (`#step-pick`) — grid of activity cards with checkboxes; search box + category `<select>` filter the visible cards (selection state is independent of filtering).
2. **Rank** (`#step-rank`) — ordered list of only the selected activities. Reorder via HTML5 drag-and-drop (`draggable` + `dragover`/`dragstart`/`dragend`) or the ↑/↓ buttons.
3. **Share** (`#step-share`) — renders the ranked list as plain text and offers:
   - **Share…** — `navigator.share`, using the Web Share API's file-sharing variant (`navigator.canShare({ files })`) to share a `.txt` file directly when supported, falling back to `title`/`text` share, then hidden entirely with a fallback message if `navigator.share` doesn't exist.
   - **Download .txt** — always available; builds a `Blob` and triggers a synthetic anchor download.

## Persistence

- `localStorage` key: `favoriteActivities.v1`
- Stores only the array of selected activity `id`s, in ranked order (order in the array *is* the rank).
- On load, the app filters out any stored ids that no longer exist in `activities.json` (e.g. if the data file changes).
- **Reset all picks** (footer button) clears the array and localStorage after a confirmation prompt.

## Notes / known quirks

- A handful of near-duplicate names exist as separate entries because they appear in two different source tables with different framing (e.g. "Puffin cruise" under Wildlife vs. "Puffin cruises" under Coastal) — this reflects the source document, not a parsing bug.
